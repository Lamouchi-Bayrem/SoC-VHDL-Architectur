# Contributing

Contributions are welcome.

## How to contribute

1. Fork the repository.
2. Create a new branch.
3. Add or improve VHDL modules, testbenches, documentation, or FPGA constraints.
4. Run the simulation scripts.
5. Open a pull request.

## Coding style

- Use clear entity and architecture names.
- Keep one VHDL entity per file.
- Add comments for control signals.
- Add a testbench for each module.
- Prefer `numeric_std` for new arithmetic code.

## Suggested contributions

- More instruction examples
- Better memory initialization
- Vivado/Quartus project scripts
- More FPGA board constraints
- UART/SPI/I2C peripherals
